Agents for Product Store
=========================

Purpose
-------
This file tells assistant agents how to work on the Product Store project (Flask).

Project context (summary)
-------------------------
- Stack: Python + Flask backend, single plain-HTML frontend. See AGENTS.md for full project context.
- Storage: in-memory Python structures (no database). Do not add a database unless explicitly requested.
- Key files: `app.py`, `index.html`, `test_validation.py`.

Agent rules and responsibilities
--------------------------------
- Preserve the project's simple architecture: no npm, no React, no build steps.
- Keep changes minimal and focused; prefer small, well-tested patches.
- Add or update tests for any behavior change. Use `pytest` and put tests alongside existing tests.
- When adding endpoints, include validation, explicit error responses, and acceptance criteria.
- If a change requires persistent storage beyond the in-memory dict, propose it and wait for explicit approval.

Workflow conventions
--------------------
- Run the Flask app locally on `http://127.0.0.1:5000` when manually verifying UI changes.
- Use Playwright for end-to-end checks when requested; use `pytest` for unit/integration tests.
- Keep frontend edits limited to `index.html` unless building a new UI is requested.

Commit & PR guidance
--------------------
- Keep commits small and focused; include tests where applicable.
- In PR descriptions, include: summary, files changed, migration notes (if any), and how to run tests.

Where to look first
-------------------
- Start with `app.py` for backend logic and existing endpoint patterns.
- Review `index.html` for UI constraints and how forms call the backend.
- Check `test_validation.py` for testing style and patterns used in the repo.

If uncertain
------------
Ask the repo maintainer before adding significant infrastructure (databases, background workers, packages).
